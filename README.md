#  «12.8. Резервное копирование баз данных — Сараев Антон».
## Задание 1
Варианты резервного копирования для различных сценариев
####  1.1. Восстановление данных за предыдущий день
Для этого случая оптимально подходит ежедневное полное резервное копирование. Каждый день создаётся полная копия базы данных, что позволяет при необходимости восстановить все данные на состояние конца предыдущего дня. Такой подход обеспечивает максимальную надёжность и простоту восстановления, хотя требует больше места для хранения и времени на выполнение бэкапа.

####  1.2. Восстановление данных за час до предполагаемой поломки
Здесь рекомендуется использовать комбинированную схему: ежедневное полное резервное копирование в сочетании с инкрементальными или дифференциальными копиями, а также журналирование транзакций (transaction log backup). Например, ночью выполняется полный бэкап, а в течение дня — инкрементальные копии (например, каждый час). Это позволяет восстановить базу на любой момент времени в пределах суток, минимизируя потери данных.

####  1.3. Моментальное переключение при поломке базы
В этом случае используется резервный сервер (standby), который постоянно синхронизируется с основным. При сбое основного сервера происходит автоматическое переключение на резервный, что обеспечивает минимальное время простоя и практически мгновенное восстановление работы. Для этого применяются технологии такие как Always On Availability Groups в MS SQL Server, Oracle Data Guard, MySQL Replication и другие.

## Задание 2
Для создания резервной копии базы данных PostgreSQL используется утилита pg_dump. Пример команды для создания дампа базы mydatabase от имени пользователя myuser:
```bash
pg_dump -U myuser -F c -f mydatabase_backup.dump mydatabase
```
* -U myuser — имя пользователя.
* -F c — формат дампа (custom, рекомендуется для гибкости).
* -f mydatabase_backup.dump — имя выходного файла.
* mydatabase — имя базы данных.
* 
Для восстановления базы из резервной копии используется pg_restore:
```bash
pg_restore -U myuser -d mydatabase -C mydatabase_backup.dump
```
